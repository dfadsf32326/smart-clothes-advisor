# Wardrobe Manager (穿搭管理) 🐾

基于 26°C 穿衣法则的智能穿搭推荐系统。

## 核心逻辑
1. **26°C 穿衣法则**：目标体感温度 = 环境温度 + 衣服保暖度 = 26°C。
2. **老板特质 (非常怕冷)**：26°C 仅作为初始基准。AI 需根据老板“冷/暖”的反馈，自动微调单品保暖分值或全局目标温度。
3. **分层穿搭 (Layering)**：针对大温差场景，强制推荐“洋葱式”穿搭（吸汗内层+保暖中层+易穿脱外层），便于随气温变化穿脱。
4. **风格维度**：单品和套装需标注风格（商务、日系、韩系、美式等），支持按风格场景推荐。
5. **成套优先**：优先推荐已保存的“成套穿搭”，无合适套装时再进行单品组合。
6. **动态微调**：支持对单品保暖度进行个性化微调。

## 目录结构
- `data/items.json`: 衣服单品仓库（含名称、分类、保暖度、图片路径等）。
- `data/outfits.json`: 成套穿搭方案（单品组合、适用场景等）。
- `config/rules.json`: 26°C 法则微调参数及保暖度标准。
- `assets/images/`: 衣服及穿搭照片存储。

## 交互指令
- **录入单品**：老板发照片或描述衣服，我来归档。
- **保存套装**：记录“上衣+下衣+配饰”的固定搭配。
- **今日穿什么**：根据定位气温，结合全天温差走势，推荐最合适的穿法。
- **穿衣微调**：觉得某件衣服推荐厚了或薄了，告诉我，我记在心里。
- **购物建议**：库存不足以应对极端天气时，我会提醒老板该买买买啦！
