# Smart Clothes Advisor (智能衣服穿搭助手) 🐾

基于 26°C 穿衣法则的智能穿搭推荐系统。

## 核心逻辑
1. **动态穿衣法则**：基础目标体感温度设定为 26°C（环境温度 + 衣服保暖度 = 目标温度）。
2. **全局温度微调优先**：老板特质是比较怕冷。当觉得冷/热时，优先通过增加/降低全局的“当前目标温度”（例如调高至27°C/28°C，或降低至24°C）来调整推荐，**严禁随意修改单件衣服本身的保暖分值**。衣服自身的保暖分值是客观且相对固定的。
3. **分层穿搭 (Layering)**：针对大温差场景，强制推荐“洋葱式”穿搭（吸汗内层+保暖中层+易穿脱外层），便于随气温变化穿脱。
4. **风格维度**：单品和套装需标注风格（商务、日系、韩系、美式等），支持按风格场景推荐。
5. **成套优先**：优先推荐已保存的“成套穿搭”，无合适套装时再进行单品组合。
6. **动态微调**：支持对单品保暖度进行个性化微调。

## 目录结构
- `data/items.json`: 衣服单品仓库（含名称、分类、保暖度、图片路径等）。
- `data/outfits.json`: 成套穿搭方案（单品组合、适用场景等）。
- `config/rules.json`: 26°C 法则微调参数及保暖度标准。
- `assets/images/`: 衣服及穿搭照片存储。

## 交互指令
- **录入单品**：老板直接把衣服图片发给我，我会自动调用大模型视觉识别图片中的衣物信息（名称、品类、颜色、保暖度、风格等），向老板展示提取结果进行确认。用户确认或修改无误后，我会自动调用脚本（`add_single_item.py`）推送到飞书多维表格，并同步到本地数据库。**注意：必须在老板明确回复确认后，才能调用脚本录入，禁止直接静默推送。**
- **保存套装**：记录“上衣+下衣+配饰”的固定搭配。
- **今日穿什么**：根据定位气温，结合全天温差走势，推荐最合适的穿法。
- **穿衣微调**：觉得推荐的衣服冷了或热了时，我会帮您动态调整“当前目标温度”（如从 26°C 提高到 28°C），衣服本身的客观保暖值保持不变。
- **购物建议**：库存不足以应对极端天气时，我会提醒老板该买买买啦！
